如何使用 Chrome 命令启动器 Steward 快速浏览网站

2018-04-30

在之前的文章里，我介绍了 Steward 的三种模式，这篇文章将介绍如何利用三种模式快速浏览网站与网页。

Steward 是开源项目，代码托管在 Github 上，从 Chrome应用商店 安装，或 离线下载，官网是oksteward.com，也有官方论坛bbs.oksteward.com。

访问整个网络 — 多命令

此功能可以在 Steward 的任意模式中使用。

书签

命令：bm

历史记录

命令: his

最常访问

命令：site

由于 Steward 会占用新标签页，因此提供 site 命令查看 Chrome 原生新标签上显示的最常访问网站列表。

直接输入链接

直接在 Steward 输入框输入你要访问的网站 URL，按 Enter / Return 键即可。

使用搜索引擎

当你的输入没有匹配到任何内容时，Steward 会列出搜索引擎列表，选择后会跳转到指定网站的搜索页面

搜索引擎设置

命令：se

站点访问 — 导航

通常要访问站点内的其它特定页面，只能手动输入链接或点击什么一下。虽然 Vimium / Surfingkeys 的 F 键能快速打开页面内的链接，但往往需要输入 2、3 个无意义的字母，感觉不太方便，而且它只能查找页面内可见区域的链接，对于不可见的就无能为力了。

此功能需要在 Steward 的页面模式中使用，即通过页面模式快捷键唤起 Steward 输入框。

站点配置

在选项 -> Websites 面板进行管理

自定义 Paths

考虑到一个网站的网站地图变化不会太大，所以想到可以为指定站点添加一些常用页面，通过 Steward 来快速访问。

比如我想添加少数派的个人主页，只需要在 Paths 处分别输入 title: 个人主页，path: /user/784469/updates ，点击 + 按钮即可。

自动提取导航

开发过程中，发现很多网站页面都是有导航区域存在的，这里面的链接通过简单地指定 css selector 就可以提取出来。

对于符合 HTML5 规范的站点/页面，默认的 nav a就基本满足；对于其它站点/页面，了解 css 的同学可以通过 Chrome DevTools 查看，不了解 css 也没关系，可以去Steward：中文社区发贴或在 Telegram 交流群中求助。

页面浏览 — 大纲/outline

当我们浏览篇幅较长的文章时，往往需要反复地快速定位到文章内的某些部分，虽然一些扩展支持自动提取页面大纲，但基本都做不到配置灵活与快速定位。而 Steward 作为一个浏览器启动器，天然地适合这份工作。

以少数派文章举例，只需要在少数派的 Website 配置里将 Outline Scope 字段填充为.article-content并保存即可。

说明

目前 Steward 的 Websites 功能还在进一步探索与完善中，后续陆续会加入类似站点配置导入、导出，站点配置分享，页面内 Buttons 点击等功能，敬请期待。
